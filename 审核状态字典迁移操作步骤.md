# 审核状态字典迁移操作步骤

本文档用于在已有数据库环境中执行审核状态字典迁移，使“审核列表”的审核状态支持：

- `ai_reviewing`：AI审核中
- `reviewing`：待人工审核，原“审核中”改名
- `ai_failed`：AI审核失败

同时为“审核资产库”的“审核资源列表”和“审核资产库详情页”新增独立审核状态字典，使资产库不再共用“审核列表管理”的审核状态。资产库审核状态包含：

- `returned`：驳回
- `approved`：审核通过
- `reviewing`：待人工审核

迁移脚本位置：

```text
backend/sql/audit_review_ai_status_migration.sql
backend/sql/audit_asset_review_status_dict_migration.sql
```

默认数据库配置来自 `backend/ruoyi-admin/src/main/resources/application-druid.yml`：

```text
host: 127.0.0.1
port: 3306
database: ry-vue
username: ruoyi
```

如果你的环境中数据库地址、库名、用户名或密码不同，请以实际部署配置为准。

## 一、迁移前准备

1. 确认代码已更新到包含以下文件的版本：

```text
backend/sql/audit_review_ai_status_migration.sql
backend/sql/audit_asset_review_status_dict_migration.sql
```

2. 确认 MySQL 服务正在运行，并且能用项目配置中的账号连接数据库。

3. 建议先停止正在进行的审核状态配置操作，避免迁移期间有人同时在后台修改字典。

4. 本迁移脚本是幂等的，可以重复执行。重复执行不会重复插入 `ai_reviewing`、`ai_failed` 和 `audit_asset_review_status` 字典项。

5. 资产库迁移脚本会把 `audit_asset_record.review_status = 'pending'` 的历史数据更新为 `reviewing`，与新的资产库三态字典保持一致。

## 二、Windows 环境迁移

以下步骤以 PowerShell 为例，假设你已经进入项目根目录。

```powershell
cd D:\TestEnvironment\PythonENV\Audit
```

如果你的项目目录不同，请替换成实际路径。

### 1. 确认 MySQL 客户端

如果 `mysql` 已经加入 `PATH`，执行：

```powershell
mysql --version
```

如果没有加入 `PATH`，可以使用 MySQL 的完整路径，例如：

```powershell
& "C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" --version
```

后续命令如果提示找不到 `mysql` 或 `mysqldump`，就改用完整路径。

### 2. 备份相关表

备份本次迁移涉及的 `sys_dict_data`、`sys_dict_type` 和 `audit_asset_record` 表。

如果 `mysqldump` 已加入 `PATH`：

```powershell
$ts = Get-Date -Format "yyyyMMdd_HHmmss"
New-Item -ItemType Directory -Force .\backup | Out-Null
mysqldump -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue sys_dict_data sys_dict_type audit_asset_record > ".\backup\audit_status_dict_$ts.sql"
```

如果没有加入 `PATH`：

```powershell
$ts = Get-Date -Format "yyyyMMdd_HHmmss"
New-Item -ItemType Directory -Force .\backup | Out-Null
& "C:\Program Files\MySQL\MySQL Server 8.0\bin\mysqldump.exe" -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue sys_dict_data sys_dict_type audit_asset_record > ".\backup\audit_status_dict_$ts.sql"
```

执行后会提示输入数据库密码。

### 3. 执行迁移 SQL

PowerShell 对 `<` 输入重定向支持不稳定，建议用 `cmd /c` 执行。

先执行“审核列表管理”的审核状态迁移。

如果 `mysql` 已加入 `PATH`：

```powershell
cmd /c "mysql -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue < backend\sql\audit_review_ai_status_migration.sql"
```

如果没有加入 `PATH`：

```powershell
cmd /c """C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe"" -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue < backend\sql\audit_review_ai_status_migration.sql"
```

再执行“审核资产库”的独立审核状态字典迁移。

如果 `mysql` 已加入 `PATH`：

```powershell
cmd /c "mysql -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue < backend\sql\audit_asset_review_status_dict_migration.sql"
```

如果没有加入 `PATH`：

```powershell
cmd /c """C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe"" -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue < backend\sql\audit_asset_review_status_dict_migration.sql"
```

执行后输入数据库密码。没有报错即表示 SQL 已执行完成。

### 4. 验证迁移结果

进入 MySQL：

```powershell
mysql -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue
```

执行：

```sql
SELECT dict_sort, dict_label, dict_value, list_class, is_default, status
FROM sys_dict_data
WHERE dict_type = 'audit_review_status'
ORDER BY dict_sort, dict_code;
```

应能看到以下关键结果：

```text
AI审核中      ai_reviewing    primary
待人工审核    reviewing       warning
AI审核失败    ai_failed       danger
```

其中 `reviewing` 的 `dict_label` 应为“待人工审核”，不再是“审核中”。

继续执行：

```sql
SELECT dict_sort, dict_label, dict_value, list_class, is_default, status
FROM sys_dict_data
WHERE dict_type = 'audit_asset_review_status'
ORDER BY dict_sort, dict_code;
```

应能看到以下三项：

```text
驳回          returned     danger
审核通过      approved     success
待人工审核    reviewing    warning
```

确认资产库历史 `pending` 状态已迁移：

```sql
SELECT COUNT(*) AS pending_asset_count
FROM audit_asset_record
WHERE review_status = 'pending';
```

结果应为 `0`。

退出 MySQL：

```sql
exit;
```

### 5. 刷新字典缓存

RuoYi 会缓存字典数据。迁移 SQL 执行完成后，需要刷新缓存，否则页面可能仍显示旧字典。

推荐方式：

1. 使用管理员账号登录系统。
2. 进入“系统管理”。
3. 进入“字典管理”。
4. 点击“刷新缓存”。
5. 浏览器刷新页面，或退出后重新登录。

如果前端仍显示旧的“审核中”，请清理浏览器缓存后重新登录。

### 6. 重启服务

如果本次迁移同时伴随后端或前端代码更新，需要重启对应服务。

Windows 开发环境中，如果服务是手动启动的，关闭原来的终端进程后重新启动：

```powershell
# 后端按你的实际启动方式重新启动

# 前端示例
cd .\ruoyi-ui
npm run dev
```

如果只是执行字典迁移，且已经在后台点击“刷新缓存”，通常不需要重启后端。

## 四、功能验证

迁移和缓存刷新完成后，登录系统验证：

1. 进入“审核列表管理”。
2. 打开“审核列表”。
3. 查看“审核状态”筛选框，应包含：

```text
AI审核中
待人工审核
AI审核失败
```

4. 列表中的“审核状态”列应能正常显示以上状态。
5. 选择“AI审核中”筛选，应能筛出最新版本 AI 任务仍处于 `waiting`、`executing` 或 `paused` 的任务。
6. AI 工作流完成后，对应任务应显示为“待人工审核”。
7. AI 工作流失败后，对应任务应显示为“AI审核失败”。
8. 进入“审核资产库”。
9. 打开“审核资源列表”。
10. 查看“审核状态”筛选框，应只包含：

```text
驳回
审核通过
待人工审核
```

11. 打开任意审核资产详情页，详情页中的“任务审核状态”也应使用以上三种资产库专用状态展示。

## 五、回滚方式

如果迁移后需要回滚，优先使用迁移前备份的 `sys_dict_data`、`sys_dict_type` 和 `audit_asset_record` 表。

Windows 示例：

```powershell
cmd /c "mysql -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue < backup\audit_status_dict_你的备份时间.sql"
```

Ubuntu 示例：

```bash
mysql -h 127.0.0.1 -P 3306 -u ruoyi -p --default-character-set=utf8mb4 ry-vue < backup/audit_status_dict_你的备份时间.sql
```

回滚后同样需要刷新字典缓存，并刷新浏览器页面。

## 六、常见问题

### 1. 执行 SQL 后页面仍显示“审核中”

原因通常是字典缓存未刷新。请进入“系统管理” -> “字典管理”，点击“刷新缓存”，然后刷新浏览器或重新登录。

### 2. 提示 Access denied

数据库用户名或密码不正确，或者当前账号没有目标库权限。请核对 `application-druid.yml` 或生产环境实际配置。

### 3. 提示 Unknown database

库名不正确。默认库名是 `ry-vue`，但生产环境可能使用了其他库名，请以实际配置为准。

### 4. 提示找不到 SQL 文件

请确认当前目录是项目根目录，并且文件存在：

```bash
ls backend/sql/audit_review_ai_status_migration.sql
ls backend/sql/audit_asset_review_status_dict_migration.sql
```

Windows PowerShell 中可执行：

```powershell
Test-Path .\backend\sql\audit_review_ai_status_migration.sql
Test-Path .\backend\sql\audit_asset_review_status_dict_migration.sql
```

### 5. 只执行 SQL 是否足够

本 SQL 只负责更新字典数据和资产库历史状态值。审核列表动态显示“AI审核中”“待人工审核”“AI审核失败”，以及审核资产库使用独立审核状态字典的逻辑还依赖后端和前端代码更新。已有环境上线时应同时部署最新代码，并按实际部署方式重启后端和前端服务。
