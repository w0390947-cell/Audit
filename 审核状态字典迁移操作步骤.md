# Ubuntu 现有库更新上传依据工作流 Prompt 操作步骤

本文档用于在 Ubuntu 部署环境中，将现有 `audit_workflow` 数据库里的“上传依据审核”工作流 Prompt 更新为新版 Prompt。

本次迁移只影响以下记录：

```text
database: audit_workflow
table: audit_workflow
workflow_code: uploaded_basis_document_audit
field: prompt_template
```

迁移脚本位置：

```text
audit-workflow-service/sql/update_uploaded_basis_prompt_20260527.sql
```

该脚本会在旧 Prompt 的“内容禁止项”和“## 输出规范”之间插入 `## 审核标准` 1-34 条，并保留原有 JSON 输出示例。脚本带幂等条件，已经包含 `## 审核标准` 时不会重复插入。

## 一、进入部署目录

以下以 `/opt/audit` 为例。如果你的项目部署目录不同，请替换为实际目录。

```bash
cd /opt/audit
```

确认迁移脚本存在：

```bash
ls -l audit-workflow-service/sql/update_uploaded_basis_prompt_20260527.sql
```

如果文件不存在，先更新代码或上传该 SQL 文件。

## 二、确认数据库连接信息

默认工作流数据库配置通常在：

```text
audit-workflow-service/src/main/resources/application.yml
```

请确认以下配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/audit_workflow
    username: audit_workflow
    password: AuditWorkflow_2026@Mysql!
```

后续命令中的主机、端口、库名、用户名请以实际配置为准。

## 三、确认 MySQL 客户端

```bash
mysql --version
mysqldump --version
```

如果提示命令不存在，先安装客户端：

```bash
sudo apt update
sudo apt install -y mysql-client
```

## 四、备份现有 Prompt

建议迁移前备份 `audit_workflow` 表，至少备份目标工作流记录。

```bash
mkdir -p backup
ts=$(date +%Y%m%d_%H%M%S)
mysqldump -h 127.0.0.1 -P 3306 -u audit_workflow -p \
  --default-character-set=utf8mb4 \
  audit_workflow audit_workflow \
  --where="workflow_code='uploaded_basis_document_audit'" \
  > "backup/uploaded_basis_prompt_${ts}.sql"
```

执行后输入数据库密码。

确认备份文件非空：

```bash
ls -lh "backup/uploaded_basis_prompt_${ts}.sql"
```

## 五、执行迁移 SQL

```bash
mysql -h 127.0.0.1 -P 3306 -u audit_workflow -p \
  --default-character-set=utf8mb4 \
  audit_workflow < audit-workflow-service/sql/update_uploaded_basis_prompt_20260527.sql
```

执行后输入数据库密码。没有报错即表示 SQL 已执行完成。

## 六、验证迁移结果

进入数据库：

```bash
mysql -h 127.0.0.1 -P 3306 -u audit_workflow -p \
  --default-character-set=utf8mb4 \
  audit_workflow
```

执行以下 SQL：

```sql
SELECT
  workflow_code,
  workflow_name,
  prompt_template LIKE '%## 审核标准%' AS has_review_standards,
  prompt_template LIKE '%JSON 对象必须符合以下结构：%' AS has_json_schema_example,
  prompt_template LIKE '%"findings": []%' AS has_empty_findings_example
FROM audit_workflow
WHERE workflow_code = 'uploaded_basis_document_audit';
```

期望结果：

```text
has_review_standards       = 1
has_json_schema_example    = 1
has_empty_findings_example = 1
```

继续检查插入位置：

```sql
SELECT
  LOCATE('内容禁止项', prompt_template) AS forbidden_pos,
  LOCATE('## 审核标准', prompt_template) AS standards_pos,
  LOCATE('## 输出规范', prompt_template) AS output_spec_pos
FROM audit_workflow
WHERE workflow_code = 'uploaded_basis_document_audit';
```

应满足：

```text
forbidden_pos < standards_pos < output_spec_pos
```

退出 MySQL：

```sql
exit;
```

## 七、是否需要重启服务

建议重启 `audit-workflow-service`。

原因是工作流定义可能会被服务缓存，重启可以确保后续新任务读取到数据库中的最新 Prompt。

如果使用 systemd：

```bash
sudo systemctl restart audit-workflow-service
sudo systemctl status audit-workflow-service --no-pager
```

如果是手动启动方式，例如：

```bash
mvn spring-boot:run
```

则停止原进程后重新启动。

## 八、回滚方式

如果迁移后需要回滚，可以用第四步生成的备份文件恢复目标记录。

```bash
mysql -h 127.0.0.1 -P 3306 -u audit_workflow -p \
  --default-character-set=utf8mb4 \
  audit_workflow < backup/uploaded_basis_prompt_你的时间戳.sql
```

恢复后建议重启 `audit-workflow-service`。

## 九、注意事项

- 该迁移只影响之后新发起的 AI 工作流任务。
- 已经完成的任务不会重新生成审核结果。
- 正在运行中的任务可能已经读取了旧 Prompt，建议在没有正在执行的 AI 审核任务时执行迁移。
- 迁移脚本可以重复执行，不会重复插入 `## 审核标准`。
